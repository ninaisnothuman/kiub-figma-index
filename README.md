# kiub-figma-index

A small searchable registry of every Figma file in the **Kiub** project (not the
archive), built so people can find designs from Slack in seconds without
guessing file names or scrolling pages.

## Layout

```
kiub-figma-index/
├── files/                      # Curated overlays (hand-edited, source of truth for descriptions)
│   ├── v0.yaml
│   ├── landing-pages-v0.yaml
│   └── cover-images.yaml
├── generated/                  # Cached page samples from the Figma API (optional)
├── scripts/
│   ├── build_index.py          # Merge overlays + live Figma state → index.json
│   └── find.py                 # Slack-style ranked lookup CLI
├── index.json                  # Flat searchable index (regenerated by build_index.py)
└── README.md
```

## Descriptor format

Every file gets one YAML overlay in `files/`. Each overlay has file-level
metadata plus a `pages:` list. For each page you write:

- `page_id` — the Figma canvas id (so the merger can pull live frame counts and a deeplink)
- `name` — your preferred human name (the index uses this, not Figma's `Desktop`/`Mobile` reuse)
- `purpose` — 1–3 sentence description of what's actually in the page
- `covers` — a flat list of search keywords (`upgrade-plan`, `tweet-card`, `billing-history`, …)
- `status` — `active` | `draft` | `experiment` | `archived` | `junk` (junk is excluded from search by default)
- Optional: `section`, `platforms`, `themes`, `states`, `tags`

The build script also auto-pulls `last_modified`, `frame_count`, real
`frame_names`, and a deeplinked first-frame node id from the Figma API.

## Building the index

```bash
python3 scripts/build_index.py
```

Reads each `files/*.yaml`, calls the Figma API for live state, merges, writes
`index.json`. Should be run on a cron (nightly is fine — overlays change rarely
and `last_modified` does the heavy lifting).

Requires `FIGMA_TOKEN` in `~/.openclaw/secrets.json` and `pyyaml`.

## Sync mechanism

The overlay YAMLs are the source of truth. READMEs in Figma are *generated*
from them and stay in sync via two scripts:

```bash
python3 scripts/sync.py drift   # cron-safe drift report; exits non-zero on drift
python3 scripts/sync.py gen     # regenerates generated/<file>_readme.js for each overlay
python3 scripts/sync.py show v0 # print the JS for one overlay (pipe into use_figma)
```

**Daily cron** (suggested):
```bash
cd ~/.openclaw/workspace/kiub-figma-index \
  && python3 scripts/build_index.py \
  && python3 scripts/sync.py gen \
  && python3 scripts/sync.py drift \
     || /path/to/post_to_slack.sh < drift.json
```

**Drift detection** flags:
- `NEW pages in Figma not in overlay` — designer added a page; add a descriptor to the YAML
- `STALE overlay entries (not in Figma anymore)` — page was deleted; remove from YAML
- `RENAMED in Figma vs overlay` — designer renamed a page; reconcile names

**Applying README updates to live Figma:**
The `generated/*_readme.js` files contain idempotent JS snippets (they clear the existing
README and redraw from the overlay). Pure Python can't write to Figma — Figma's REST API
is read-only — so the apply step happens via the Figma Plugin API. Two paths:
1. Ask Claude: *"sync the v0 README from overlay"*. Claude pipes the generated JS into
   the Figma MCP `use_figma` tool. Verified working end-to-end.
2. Paste the generated JS into a Figma plugin you author yourself.

The generator is deterministic — running `gen` followed by Claude-applying the JS produces
a bit-identical README every time.

## Querying

```bash
python3 scripts/find.py "upgrade plan modal"
python3 scripts/find.py "slack reaction tasks" --limit 5
python3 scripts/find.py "billing history" --json   # JSON output for bots
python3 scripts/find.py "junk wireframes" --include-junk
```

Ranking is BM25 over the search blob, plus a `+1.5` per query token that
exact-matches a `covers`/`tags` entry, multiplied by a status boost
(`active=1.0`, `draft=0.7`, `experiment=0.5`, `archived=0.4`, `junk=0.05`).

Returned per result: file, section, page name, 1-line purpose, tags, and a
Figma deep link that opens directly on the relevant frame.

## Hooking it into Slack

The same `find.py` powers a `/find-design` slash command. The handler is ~30
lines:

```python
results = search(text, limit=3)
blocks = [render_block(p) for _, p in results]
```

Each `render_block` becomes a Slack section block with the file thumbnail
(which the build script can be extended to upload), the deeplink as a button,
and the 1-line purpose as the body.

## Conventions for the underlying Figma files

These are the cleanups that compound the registry's value:

1. Rename every `Junk` page to `🗑️ Junk — DO NOT USE`. The index still flags
   them as `status: junk` so they only surface with `--include-junk`.
2. Stop reusing bare `Desktop` / `Mobile` page names. Use `Desktop · Settings`,
   `Mobile · Onboarding`, etc. The overlay's `name` field is what the index
   actually uses, but matching the Figma name avoids confusion.
3. Prefix `html.to.design` and other auto-generated experiment pages with
   `🧪 EXPERIMENT —`. They're already marked `status: experiment`.
4. Add a single `📄 README` page at the top of every Figma file mirroring the
   overlay's purpose / status / "what's not in this file" pointers, so
   designers updating the file see the same descriptor in-context and don't
   let it drift.
